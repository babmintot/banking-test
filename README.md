# banking-test

-----------------------------------------------------------------------------------------------

Описание проекта
В рамках задания выполнено тестирование веб-сервиса перевода денежных средств с банковского счёта на карту.

Реализованы:
ручное тестирование;
составление тестовой документации;
автоматизация тестирования с использованием Selenium;
настройка CI через GitHub Actions.

-----------------------------------------------------------------------------------------------
Ручное тестирование

Проведено тестирование следующих сценариев:

успешный перевод средств;
отказ при недостатке средств;
проверка граничного значения суммы;
валидация номера карты (длина < 16, > 16);
валидация суммы (отрицательная, пустая);
проверка расчёта комиссии.

-----------------------------------------------------------------------------------------------

Найденные дефекты:

Ошибка обработки граничного значения суммы;
Некорректная валидация номера карты (длина > 16);
Отсутствие валидации отрицательной суммы.
Автоматизированное тестирование
Реализованы Selenium-тесты для следующих сценариев:

-----------------------------------------------------------------------------------------------

Позитивные сценарии:

успешный перевод;
недостаточно средств;
отрицательная сумма;
пустая сумма;
проверка длины карты (< 16);
проверка комиссии.
Дефектные сценарии:

граничное значение суммы (BUG);
номер карты > 16 цифр (BUG).

-----------------------------------------------------------------------------------------------

Локальный запуск проекта
Требования
Для запуска необходимо установить:

Node.js (версия 18+)
npm
Python 3
Google Chrome (последняя версия)
-----------------------------------------------------------------------------------------------
Запуск тестируемого сервиса
cd dist
python3 -m http.server 8000

Открыть в браузере:
http://localhost:8000/?balance=30000&reserved=20001
-----------------------------------------------------------------------------------------------

Запуск автотестов
Установка зависимостей:

cd autotests
npm install

Запуск всех тестов:

npm run test-all-local

Запускаются:

позитивные тесты
дефектные тесты
Запуск только позитивных тестов:

npm run test-positive

Запуск отдельных тестов:

npm run test-success
npm run test-defect-boundary
npm run test-defect-card

Важно
Перед запуском автотестов убедитесь, что:

сервер запущен (python3 -m http.server 8000)
приложение доступно по адресу http://localhost:8000
Иначе возникнет ошибка:
ERR_CONNECTION_REFUSED

-----------------------------------------------------------------------------------------------

CI (GitHub Actions)
Настроен автоматический запуск тестов при:

push
pull request
CI выполняет:

установку зависимостей
запуск тестируемого сервиса
запуск Selenium-тестов
Особенность:
Итоговая сборка является КРАСНОЙ (FAILED) намеренно, так как автотесты воспроизводят найденные дефекты:

ошибка граничного значения суммы
некорректная валидация номера карты

-----------------------------------------------------------------------------------------------
Итоги
В ходе выполнения задания:

разработаны тест-кейсы
выявлены и задокументированы дефекты
реализованы автотесты
настроен CI
